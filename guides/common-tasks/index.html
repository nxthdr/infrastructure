    Common Tasks¶

    This guide covers frequently performed infrastructure operations.

    Container Management¶

    Update Container Configuration¶

    When you need to change a container's configuration files:

    1. Edit the template: 

      vim templates/config/core/coreams01/{service}/config/{file}.j2

    2. Deploy: 

      make apply

    3. Restart container (if Terraform didn't detect changes): 

      ssh nxthdr@ams01.core.infra.nxthdr.dev
docker restart {container_name}

    Update Container Image Version¶

    To update a Docker image to a new version:

    1. Edit Terraform file: 

      vim terraform/{hostname}.tf

    2. Update the image tag: 

      resource "docker_image" "prometheus" {
  name = "prom/prometheus:v2.48.0"  # Change version here
  provider = docker.coreams01
}

    3. Apply changes: 

      make apply

    Terraform will pull the new image and recreate the container.

    Restart a Container¶

    ssh nxthdr@ams01.core.infra.nxthdr.dev
docker restart {container_name}

    View Container Logs¶

    ssh nxthdr@ams01.core.infra.nxthdr.dev
docker logs -f {container_name}

    Execute Commands in Container¶

    ssh nxthdr@ams01.core.infra.nxthdr.dev
docker exec -it {container_name} {command}

    Example for Redpanda: 

    docker exec -it redpanda rpk topic list

    Network Configuration¶

    Update BIRD (BGP) Configuration¶

    1. Edit configuration: 

      vim networks/{hostname}/bird/bird.conf

    2. Sync to server: 

      make sync-bird

    Enter BECOME password when prompted.

    1. Verify: 
      ssh nxthdr@ams01.core.infra.nxthdr.dev
sudo birdc show status
sudo birdc show protocols

    Update WireGuard Configuration¶

    1. Edit configuration: 

      vim networks/{hostname}/wireguard/{interface}.conf

    2. Sync to server: 

      make sync-wireguard

    Enter BECOME password when prompted.

    1. Verify: 
      ssh nxthdr@ams01.core.infra.nxthdr.dev
sudo wg show

    Secrets Management¶

    View Secrets¶

    make edit-secrets

    This opens secrets/secrets.yml in your editor.

    Add a New Secret¶

    1. Edit secrets file: 

      make edit-secrets

    2. Add your secret: 

      new_service_password: "your_secure_password"

    3. Save and close the editor

    4. Use in templates: 

      password={{ secrets.new_service_password }}

    Rotate a Secret¶

    1. Update the secret: 

      make edit-secrets

    2. Change the value in the editor

    3. Re-render and deploy: 

      make apply

    4. Restart affected services: 

      ssh nxthdr@ams01.core.infra.nxthdr.dev
docker restart {affected_container}

    Template Management¶

    Add a New Configuration File¶

    1. Create template: 

      vim templates/config/core/coreams01/{service}/config/newfile.conf.j2

    2. Add content with Jinja2 variables as needed

    3. Deploy: 

      make apply

    The file will be synced to /home/nxthdr/{service}/config/newfile.conf on the server.

    Test Template Rendering¶

    Render without deploying:

    make render

    Check output:

    cat .rendered/{hostname}/{service}/config/{file}

    Available Template Variables¶

    In any Jinja2 template, you have access to:

    • {{ inventory_hostname }} - Current host (e.g., coreams01)
    • {{ ansible_host }} - Server FQDN
    • {{ ansible_user }} - SSH user
    • All group vars from inventory/inventory.yml
    • All host vars from inventory/inventory.yml
    • All secrets from secrets/secrets.yml via {{ secrets.key_name }}

    Deployment Workflows¶

    Deploy Everything¶

    Full deployment (render, sync, apply):

    make apply

    Deploy Only Configuration Changes¶

    If you only changed configs (no Terraform changes):

    make sync-config

    Then restart affected containers manually.

    Deploy Only Terraform Changes¶

    If you only changed Terraform files:

    terraform -chdir=./terraform apply

    Dry Run / Preview Changes¶

    Preview rendering: 

    make render
git diff .rendered/

    Preview Ansible sync: 

    ansible-playbook -i inventory/ playbooks/sync-config.yml --check

    Preview Terraform changes: 

    terraform -chdir=./terraform plan

    Monitoring and Debugging¶

    Check Service Health¶

    Access Grafana dashboards: 

    https://grafana.nxthdr.dev

    View System Logs¶

    ssh nxthdr@ams01.core.infra.nxthdr.dev
sudo journalctl -u docker -f

    Check Docker Network¶

    ssh nxthdr@ams01.core.infra.nxthdr.dev
docker network ls
docker network inspect dmz
docker network inspect backend

    Verify Container Connectivity¶

    Test if a container can reach another:

    ssh nxthdr@ams01.core.infra.nxthdr.dev
docker exec {container} ping -c 3 {other_container}

    Backup and Recovery¶

    Backup Configuration¶

    All configuration is in git:

    git pull  # Get latest
git log   # Review history

    Backup Data Volumes¶

    Important data volumes should be backed up regularly:

    ssh nxthdr@ams01.core.infra.nxthdr.dev
# Backup example for ClickHouse
sudo tar -czf clickhouse-backup-$(date +%Y%m%d).tar.gz /home/nxthdr/clickhouse/data/

    Restore from Backup¶

    1. Stop the affected container
    2. Restore data from backup
    3. Restart the container
    ssh nxthdr@ams01.core.infra.nxthdr.dev
docker stop clickhouse
sudo tar -xzf clickhouse-backup-20241005.tar.gz -C /
docker start clickhouse

    Emergency Procedures¶

    Rollback a Change¶

    If a deployment causes issues:

    1. Revert git commit: 

      git revert HEAD

    2. Re-deploy: 

      make apply

    Restart All Containers¶

    ssh nxthdr@ams01.core.infra.nxthdr.dev
docker restart $(docker ps -q)

    Warning

    This will cause brief downtime for all services.

    Emergency Access¶

    If SSH is not working, use provider console:

    • Scaleway: https://console.scaleway.com/
    • Vultr: https://my.vultr.com/
    • iFog: Contact support

    Maintenance Tasks¶

    Update System Packages¶

    ssh nxthdr@ams01.core.infra.nxthdr.dev
sudo apt update && sudo apt upgrade -y

    Clean Docker Resources¶

    Remove unused images and containers:

    ssh nxthdr@ams01.core.infra.nxthdr.dev
docker system prune -a

    Check Disk Space¶

    ssh nxthdr@ams01.core.infra.nxthdr.dev
df -h
du -sh /home/nxthdr/*
