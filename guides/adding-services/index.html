    Adding New Services¶

    This guide walks through adding a new service to the nxthdr infrastructure.

    Overview¶

    Adding a new service involves:

    1. Creating configuration templates
    2. Adding Terraform resources
    3. Deploying the service
    4. (Optional) Configuring proxy/certificates

    Example: Adding a New Service¶

    Let's add a hypothetical "metrics-exporter" service to the core server.

    Step 1: Create Configuration Directory¶

    Create a directory for your service configuration:

    mkdir -p templates/config/core/coreams01/metrics-exporter/config

    Step 2: Add Configuration Files¶

    Create configuration files (as Jinja2 templates if needed):

    vim templates/config/core/coreams01/metrics-exporter/config/config.yml.j2

    Step 3: Add Terraform Resources¶

    Edit the Terraform file for your target host:

    vim terraform/coreams01.tf

    Add the Docker image resource:

    resource "docker_image" "metrics_exporter" {
  name = "metrics-exporter:v1.0.0"
  provider = docker.coreams01
}

    Add the container resource:

    resource "docker_container" "metrics_exporter" {
  image = docker_image.metrics_exporter.image_id
  name  = "metrics-exporter"
  provider = docker.coreams01
  restart = "unless-stopped"

  # Logging
  log_driver = "json-file"
  log_opts = {
    tag = "{{.ImageName}}|{{.Name}}|{{.ImageFullID}}|{{.FullID}}"
  }

  # DNS servers
  dns = ["2a00:1098:2c::1", "2a00:1098:2b::1"]

  # Network configuration
  network_mode = "bridge"
  networks_advanced {
    name = docker_network.backend.name
    ipv6_address = "2a06:de00:50:cafe:10::50"  # Choose unused IP
  }

  # Volumes
  volumes {
    container_path = "/etc/metrics-exporter/config.yml"
    host_path = "/home/nxthdr/metrics-exporter/config/config.yml"
  }

  volumes {
    container_path = "/var/lib/metrics-exporter"
    host_path = "/home/nxthdr/metrics-exporter/data"
  }

  # Environment variables
  env = [
    "LOG_LEVEL=info",
    "TZ=UTC"
  ]

  # Expose ports (if needed for DMZ access)
  # ports {
  #   internal = 9100
  #   external = 9100
  #   protocol = "tcp"
  # }
}

    Step 4: Deploy the Service¶

    Deploy the new service:

    make apply

    This will: 1. Render the configuration template 2. Sync files to the server 3. Create the Docker container

    Step 5: Verify Deployment¶

    Check that the container is running:

    ssh nxthdr@ams01.core.infra.nxthdr.dev
docker ps | grep metrics-exporter
docker logs metrics-exporter

    Adding to IXP or VLT Servers¶

    For IXP or VLT servers, the process is slightly different since they use shared templates.

    Step 1: Create Shared Configuration¶

    Create configuration in the shared group directory:

    mkdir -p templates/config/ixp/metrics-exporter/config
vim templates/config/ixp/metrics-exporter/config/config.yml.j2

    This configuration will be used by all IXP servers.

    Step 2: Update Terraform Template¶

    Edit the Terraform template for the group:

    vim templates/terraform/ixp.tf.j2

    Add your service resources using Jinja2 variables:

    resource "docker_image" "metrics_exporter" {
  name = "metrics-exporter:v1.0.0"
  provider = docker.{{ inventory_hostname }}
}

resource "docker_container" "metrics_exporter" {
  image = docker_image.metrics_exporter.image_id
  name  = "metrics-exporter"
  provider = docker.{{ inventory_hostname }}
  restart = "unless-stopped"

  # ... rest of configuration
}

    Step 3: Deploy¶

    make apply

    This will deploy to all IXP servers.

    Exposing Service via Proxy¶

    If your service needs HTTPS access:

    1. Add to DMZ network in Terraform: 

      networks_advanced {
  name = docker_network.dmz.name
  ipv6_address = "2a06:de00:50:cafe:100::50"
}

    2. Update Caddyfile (templates/config/core/coreams01/proxy/config/Caddyfile.j2): 

      metrics.nxthdr.dev {
    reverse_proxy [2a06:de00:50:cafe:100::50]:9100
    tls {
        dns cloudflare {env.CLOUDFLARE_API_TOKEN}
    }
}

    3. Deploy: make apply (certificates generated automatically)

    Adding Secrets¶

    If your service needs secrets:

    Step 1: Add to Vault¶

    make edit-secrets

    Add your secrets:

    metrics_exporter:
  api_key: "your-secret-key"
  db_password: "your-db-password"

    Step 2: Use in Templates¶

    Reference in your Jinja2 templates:

    api_key: {{ secrets.metrics_exporter.api_key }}
database:
  password: {{ secrets.metrics_exporter.db_password }}

    Service Dependencies¶

    If your service depends on another service:

    Using Docker Networks¶

    Services on the same Docker network can communicate by container name:

    # In metrics-exporter config
database:
  host: clickhouse  # Container name
  port: 9000

    Using depends_on (Terraform)¶

    resource "docker_container" "metrics_exporter" {
  # ... other config

  depends_on = [
    docker_container.clickhouse
  ]
}

    Persistent Data¶

    For services that need persistent storage:

    Create Data Directory¶

    The directory will be created automatically when you sync configs, but you can also create it manually:

    ssh nxthdr@ams01.core.infra.nxthdr.dev
mkdir -p /home/nxthdr/metrics-exporter/data
chown 1001:1001 /home/nxthdr/metrics-exporter/data  # Adjust UID/GID as needed

    Mount Volume¶

    In Terraform:

    volumes {
  container_path = "/var/lib/metrics-exporter"
  host_path = "/home/nxthdr/metrics-exporter/data"
}

    Health Checks¶

    Add health checks to monitor service status:

    resource "docker_container" "metrics_exporter" {
  # ... other config

  healthcheck {
    test = ["CMD", "curl", "-f", "http://localhost:9100/health"]
    interval = "30s"
    timeout = "3s"
    retries = 3
    start_period = "10s"
  }
}

    Monitoring Integration¶

    Add to Prometheus¶

    Edit Prometheus configuration:

    vim templates/config/core/coreams01/prometheus/config/prometheus.yml.j2

    Add scrape config:

    scrape_configs:
  - job_name: 'metrics-exporter'
    static_configs:
      - targets: ['metrics-exporter:9100']
        labels:
          service: 'metrics-exporter'
          environment: 'production'

    Add Grafana Dashboard¶

    1. Create dashboard JSON
    2. Place in templates/config/core/coreams01/grafana/dashboards/
    3. Deploy with make apply

    Resource Limits¶

    Set CPU and memory limits:

    resource "docker_container" "metrics_exporter" {
  # ... other config

  memory = 512  # MB
  memory_swap = 1024  # MB

  # CPU shares (relative weight)
  cpu_shares = 1024
}

    Complete Example¶

    Here's a complete example for adding a Redis cache:

    # Docker image
resource "docker_image" "redis" {
  name = "redis:7-alpine"
  provider = docker.coreams01
}

# Docker container
resource "docker_container" "redis" {
  image = docker_image.redis.image_id
  name  = "redis"
  provider = docker.coreams01
  restart = "unless-stopped"

  log_driver = "json-file"
  log_opts = {
    tag = "{{.ImageName}}|{{.Name}}|{{.ImageFullID}}|{{.FullID}}"
  }

  dns = ["2a00:1098:2c::1", "2a00:1098:2b::1"]

  network_mode = "bridge"
  networks_advanced {
    name = docker_network.backend.name
    ipv6_address = "2a06:de00:50:cafe:10::60"
  }

  volumes {
    container_path = "/data"
    host_path = "/home/nxthdr/redis/data"
  }

  volumes {
    container_path = "/usr/local/etc/redis/redis.conf"
    host_path = "/home/nxthdr/redis/config/redis.conf"
  }

  command = ["redis-server", "/usr/local/etc/redis/redis.conf"]

  memory = 256

  healthcheck {
    test = ["CMD", "redis-cli", "ping"]
    interval = "10s"
    timeout = "3s"
    retries = 3
  }
}

    Checklist¶

    When adding a new service, verify:

    • Configuration templates created
    • Terraform resources added
    • Secrets added to vault (if needed)
    • Docker network configured correctly
    • Volumes configured for persistent data
    • Health checks configured
    • Monitoring/logging configured
    • Proxy configuration added (if public-facing)
    • DNS records created (if public-facing)
    • Certificates obtained (if HTTPS)
    • Service tested and verified
    • Documentation updated
